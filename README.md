# Portfolio RAG

A Retrieval-Augmented Generation (RAG) application built with Streamlit that allows visitors to ask natural language questions about a developer portfolio. Answers are generated by Gemini 2.5 Flash using context retrieved from portfolio documents via a hybrid dense + sparse retrieval pipeline with cross-encoder reranking.

---

## Features

- **Hybrid Retrieval** — combines dense FAISS semantic search, TF-IDF sparse retrieval, and BM25 for high-quality candidate selection
- **Cross-Encoder Reranking** — `ms-marco-MiniLM-L-6-v2` reranks the top candidates before passing them to the LLM
- **Query Expansion** — synonym mapping and pseudo-relevance feedback improve recall on portfolio-specific terminology
- **Near-Duplicate Filtering** — cosine similarity deduplication ensures diverse, non-redundant context chunks
- **Gemini 2.5 Flash Generation** — fast, high-quality answers grounded in your documents
- **Daily Quota Tracking** — persisted JSON-based quota management respects the Gemini free tier (20 RPD)
- **Dark Trading-Platform UI** — custom Streamlit theme built with IBM Plex fonts and a cyan-on-dark palette

---

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | Streamlit |
| Embeddings | `sentence-transformers/all-MiniLM-L6-v2` |
| Vector Index | FAISS (`IndexFlatIP`) |
| Sparse Retrieval | TF-IDF (scikit-learn) + BM25 (`rank_bm25`) |
| Reranker | `cross-encoder/ms-marco-MiniLM-L-6-v2` |
| Generation | Google Gemini 2.5 Flash |
| Document Parsing | `pypdf`, `beautifulsoup4` |

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/joshlg18/portfoliollm.git
cd portfoliollm
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Add your documents

Place your portfolio documents (`.pdf`, `.html`, or `.md`) into the `docs/` folder. Each file should represent a distinct project or section of your portfolio.

### 4. Configure your API key

Create `.streamlit/secrets.toml`:

```toml
GEMINI_API_KEY = "your-api-key-here"
```

Get a free API key at [Google AI Studio](https://aistudio.google.com).

### 5. Run the app

```bash
streamlit run app.py
```

## Deployment on Streamlit Cloud

1. Push your repository to GitHub (ensure `docs/` is included and `secrets.toml` is in `.gitignore`)
2. Go to [share.streamlit.io](https://share.streamlit.io) and connect your repository
3. Set `app.py` as the main module
4. Add `GEMINI_API_KEY` under **Settings → Secrets** in the Streamlit Cloud dashboard

### `requirements.txt`

```
streamlit
scikit-learn
beautifulsoup4

```

> **Note:** Use `faiss-cpu` — the plain `faiss` package is not available on PyPI and will cause deployment to fail.

## Configuration

Key constants at the top of `app.py`:

| Constant | Default | Description |
|---|---|---|
| `CHUNK_WORDS` | `800` | Words per chunk — sized to capture full project documents |
| `CHUNK_OVERLAP_W` | `100` | Overlap words between chunks |
| `FINAL_K` | `6` | Chunks passed to Gemini after reranking |
| `DENSE_CANDIDATES` | `64` | Candidates retrieved from FAISS before fusion |
| `FINAL_POOL` | `32` | Candidates fed to the cross-encoder |
| `DAILY_QUOTA` | `20` | Max Gemini API calls per day (free tier limit) |
| `DEDUP_COS_THRESH` | `0.90` | Cosine similarity threshold for deduplication |

## How It Works

```
Query
  │
  ├─ BM25 first-pass → pseudo-relevance feedback titles
  ├─ Query expansion (synonyms + PRF titles)
  │
  ├─ Dense FAISS retrieval (top 64)
  ├─ Sparse TF-IDF + BM25 retrieval
  ├─ Hybrid score fusion (0.50 dense · 0.30 sparse · 0.20 BM25)
  │
  ├─ Cross-encoder reranking (top 32 → scored pairs)
  ├─ Near-duplicate filtering (cosine dedup)
  │
  └─ Top-6 chunks → Gemini 2.5 Flash → Answer
```

## Quota Management

The app tracks daily Gemini API usage in `.gemini_quota.json` at the project root. The count resets automatically each day. The remaining quota is displayed in the sidebar as a progress bar. Once the daily limit is reached, queries are blocked with a clear message until the following day.

---

## Limitations

- **20 requests per day** on the Gemini free tier — suitable for demonstration and light use
- **No persistent memory** — each query is independent with no conversation history
- **Local embedding model** — first boot takes 30–60 seconds to load and index documents; subsequent queries are fast
